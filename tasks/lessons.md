# SilverPoint Lessons

## Active Rules

- Use `tasks/todo.md` for non-trivial planning before implementation.
- Check in after writing the plan and before starting implementation work.
- Keep changes small, verified, and committed with concise one-line messages.
- Use subagents for focused exploration when work has multiple architectural tracks.
- If a tool or implementation path fails in a way that changes the plan, stop and re-plan instead of pushing through blindly.
